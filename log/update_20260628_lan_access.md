# 🌐 局域网访问功能 — 更新日志 2026-06-28

## 目标

实现同一局域网内的其他设备（手机、平板、其他电脑）通过浏览器访问项目 Flask Web 服务。

---

## 技术原理

### 为什么网上邻居可以互相访问

```
┌─────────────────────────────────────────────────────────────────────┐
│                        局域网 (LAN)                                  │
│                       192.168.x.0/24                                │
│                                                                     │
│   ┌──────────────┐        ┌──────────────┐        ┌──────────────┐  │
│   │  宿主机 (Win)  │        │  手机/平板    │        │  其他电脑     │  │
│   │ IP: 192.168.  │        │ IP: 192.168.  │        │ IP: 192.168.  │  │
│   │      x.x      │        │      x.y      │        │      x.z      │  │
│   │              │        │              │        │              │  │
│   │ Flask :5000  │◄──────►│ 浏览器访问    │◄──────►│ 浏览器访问    │  │
│   │ 0.0.0.0 监听 │  HTTP  │ http://x.x   │  HTTP  │ http://x.x   │  │
│   │              │        │   :5000      │        │   :5000      │  │
│   └──────────────┘        └──────────────┘        └──────────────┘  │
│                                                                     │
│   前提条件:                                                          │
│   1. Flask 绑定 0.0.0.0 (所有网卡) — ✅ 项目已配置                    │
│   2. 防火墙放行端口 5000          — ✅ setup_firewall.bat             │
│   3. 同一子网                     — 默认满足                          │
└─────────────────────────────────────────────────────────────────────┘
```

### 关键配置项

| 配置 | 值 | 含义 |
|------|-----|------|
| `FLASK_HOST` | `0.0.0.0` | 监听本机**所有**网络接口（127.0.0.1 + LAN IP + 其他） |
| `FLASK_PORT` | `5000` | HTTP 服务端口 |
| 防火墙规则 | TCP 5000 Inbound | 允许外部设备通过 5000 端口与 Flask 通信 |

### 为什么 0.0.0.0 不是 127.0.0.1

```
127.0.0.1 (localhost)  → 只监听回环网卡，外部设备无法访问
0.0.0.0 (all)         → 监听所有网卡，外部设备可通过任意 IP 访问

实际结果:
  http://localhost:5000       ← 本机访问 ✅
  http://127.0.0.1:5000       ← 本机访问 ✅
  http://192.168.x.x:5000     ← 局域网访问 ✅ (0.0.0.0 才行)
```

### IP 检测算法

```python
# 策略1: 扫描本机所有网卡
socket.getaddrinfo(hostname, None, socket.AF_INET)
→ 过滤 127.x.x.x (回环)
→ 过滤 169.254.x.x (Windows APIPA 自动配置)
→ 保留 192.168.x.x / 10.x.x.x / 172.16-31.x.x (私有地址段)

# 策略2 (备选): UDP 连接探测
socket.connect(已知可达的IP) → getsockname() 获取实际出口IP
→ 连接到 MySQL 虚拟机 (192.168.116.128) 探测本机实际使用的 IP
```

## 使用步骤

### Windows (宿主机)

```bash
# 1. 配置防火墙 (只需一次，需管理员权限)
右键 setup_firewall.bat → 以管理员身份运行

# 2. 启动项目
.venv_py38\Scripts\activate
python main.py --web

# 3. 记下控制台输出的 IP 地址，分享给局域网用户
```

### Linux (虚拟机)

```bash
# 1. 配置防火墙
sudo bash setup_firewall.sh

# 2. 启动项目
python main.py --web
```

### 其他设备访问

```
手机/平板/其他电脑 → 打开浏览器 → 输入 http://宿主机的IP:5000
```

## 依赖

**无需额外安装**。仅使用 Python 内置 `socket` 模块和操作系统防火墙工具。

## 注意事项

1. **防火墙是主因** — Flask 已正确绑定 `0.0.0.0`，如果局域网无法访问，99% 是防火墙阻挡
2. **IP 可能变化** — DHCP 分配的 IP 可能在重启后改变，每次启动看控制台输出的实际 IP
3. **前端 CDN 依赖** — 访问设备也需要互联网连接（ECharts/Leaflet 从 CDN 加载）
4. **MySQL 必须可达** — 局域网设备访问的页面仍会通过宿主机后端查询 MySQL，不直接连数据库
